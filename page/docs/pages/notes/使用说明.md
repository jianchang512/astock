# 使用说明

本文档记录 qlib csi300 score 静态文档站的使用方法，包括本地生成与 GitHub Pages 部署。

---

## 一、系统概述

基于 **VitePress** 的静态网站，主要功能：

- 自动展示 `qlib_score_csv` 子目录下的 CSV 和 MD 文件
- 笔记与随笔：在 `pages/notes`、`pages/essays` 下新建 `.md` 即可，运行 `gen` 后自动生成索引
- 顶部导航：首页、笔记、随笔、关于

---

## 二、本地生成静态网页

### 1. 安装依赖

```bash
cd page
npm install
```

### 2. 生成页面

扫描 `qlib_score_csv` 和 `pages/notes`、`pages/essays`，生成/更新页面：

```bash
npm run gen
```

### 3. 本地开发预览

```bash
npm run docs:dev
```

浏览器访问 `http://localhost:5173`（端口可能不同，以终端输出为准）。

### 4. 构建静态站点

```bash
npm run docs:build
```

构建产物在 `page/docs/.vitepress/dist/`，可直接用任意静态服务器托管。

### 5. 本地预览构建结果

```bash
npm run docs:preview
```

## 三、部署到 GitHub Pages

### 方式一：使用 GitHub Actions（推荐）

1. **修改工作流**：将 `.github/workflows/deploy_page.yml` 中的 MkDocs 步骤改为 VitePress：

```yaml
name: 部署 VitePress 网页

on:
  workflow_dispatch:
  push:
    branches: [ qlib_score ]
    paths:
      - 'qlib_score_csv/**'
      - 'page/**'
  workflow_run:
    workflows: ["Analysis"]
    types:
      - completed

permissions:
  contents: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: 检出代码
        uses: actions/checkout@v4
        with:
          ref: qlib_score
          fetch-depth: 0

      - name: 设置 Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: page/package-lock.json

      - name: 构建 VitePress
        run: |
          cd page
          npm ci
          npm run docs:build

      - name: 部署到 GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./page/docs/.vitepress/dist
```

2. **启用 GitHub Pages**：仓库 → Settings → Pages → Source 选择 **GitHub Actions**。

3. **触发部署**：推送到 `qlib_score` 分支，或手动运行 workflow。

### 方式二：本地构建后手动上传

```bash
cd page
npm run docs:build
# 将 docs/.vitepress/dist 目录内容上传到 gh-pages 分支或作为静态资源发布
```

## 四、常用命令速查

| 命令 | 说明 |
|------|------|
| `npm run gen` | 扫描数据并生成页面 |
| `npm run docs:dev` | 本地开发（含 gen） |
| `npm run docs:build` | 构建静态站点 |
| `npm run docs:preview` | 预览构建结果 |

## 五、添加笔记与随笔

1. 在 `page/docs/pages/notes/` 或 `page/docs/pages/essays/` 下新建 `.md` 文件
2. 运行 `npm run gen`
3. 新文章会自动出现在对应列表中
